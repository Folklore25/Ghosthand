# Phase 25.5: Current-State Correction Pass For 1.4.0 - Context

**Gathered:** 2026-04-04
**Status:** Ready for planning
**Source:** Direct strict current-state correction brief after the accepted 1.4.0 baseline review

<domain>
## Phase Boundary

This correction pass owns exactly five bounded workstreams:
- truthful `/notify` cancellation symmetry
- screenshot or MediaProjection foreground-service contract alignment
- main-source wildcard-import cleanup
- contained readability or ownership extractions in the worst oversized classes
- continued test-package convergence plus focused regression coverage

This pass does not own:
- new user-facing features
- root or libsu or 2.0 work
- observation-plane redesign
- capability-plane redesign
- permission-flow redesign
- route growth
- 1.4.1 optimization work
</domain>

<decisions>
## Locked Decisions

- Treat the current review findings as binding. This is a correction pass against the current baseline, not a baseline-restoration debate.
- Keep the accepted 1.4.0 architecture baseline intact. Do not move domain-owned classes back into the root package and do not weaken `/commands`, `/state`, or `/capabilities`.
- Fix notification cancellation truthfulness by making cancel behavior symmetric with post behavior for tagged notifications while keeping the external `/notify` contract backward-compatible.
- Do not remove a buffered notification entry unless cancellation matches the posted notification identity.
- Align the screenshot or MediaProjection runtime contract across manifest declarations, foreground-service start flags, provider comments, and capability claims without reworking permission UX.
- Remove wildcard imports from main Kotlin source project-wide. This cleanup is mandatory maintainability work, not optional polish.
- Any coordinator cleanup in this pass must stay contained: extract obvious helpers or collaborators from the largest remaining coordination-heavy files without turning the pass into a redesign.
- Continue test-package convergence only where ownership is obvious, and preserve test coverage while moving root-flat tests into domain-owned packages.
- Keep the API surface stable unless a tiny backward-compatible clarification is strictly necessary.
- Preserve the loopback-only local server model and avoid speculative capability or route expansion.
</decisions>

<specifics>
## Specific Ideas

- Notification cancel currently uses `NotificationManager.cancel(id)` while post uses `NotificationManager.notify(tag, id, notification)`, so tagged notification cancellation is not truthful today.
- The foreground service is still declared and started as `dataSync` only even though screenshot capture relies on MediaProjection and the provider comments already describe a media-projection foreground-service requirement.
- Main Kotlin source still has widespread wildcard-import residue, including self-package star imports and cross-project star imports that obscure ownership. Current local inspection shows 89 main-source Kotlin files still contain wildcard imports.
- The largest remaining coordination-heavy files are still substantial: `GhostCoreAccessibilityService.kt`, `GhostAccessibilityService.kt`, `AccessibilityNodeFinder.kt`, `StateCoordinator.kt`, `RuntimeStateStore.kt`, and `GhosthandCommandCatalogRoutes.kt`.
- Sixteen root-flat unit tests still remain in `app/src/test/java/com/folklore25/ghosthand`, so test ownership convergence is not yet finished.
- Regression coverage must explicitly prove the notification truthfulness fix and should add a contract-sensitive screenshot or foreground-service drift guard where useful.
</specifics>
