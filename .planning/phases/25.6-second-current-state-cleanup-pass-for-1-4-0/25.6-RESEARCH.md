## Execution Risks

### 1. Wildcard-import cleanup to zero

- The remaining UI files still carry the same broad star-import residue pattern, so the work is largely mechanical.
- The main risk is reintroducing compile breaks if imports are removed without restoring the small set of real explicit cross-package dependencies.
- Because the remaining pollution is concentrated in 16 files, this should be finished completely rather than incrementally.

### 2. One more contained extraction pass

- `GhostCoreAccessibilityService.kt` and `GhostAccessibilityService.kt` are still large, but prior passes already extracted some shared helpers.
- The remaining safe extraction opportunities are likely lifecycle blocks, screenshot-dispatch helpers, or shared dispatch logic that can move into adjacent support files without changing service behavior.
- `StateCoordinator.kt` should lose another obvious coordination cluster, but this must stay inside the existing ownership model instead of inventing a new subsystem.

### 3. Further test-package convergence

- The remaining root-flat tests split between clearly domain-owned UI/state tests and truly cross-domain or generic tests.
- `MaintainabilityConvergencePackageOwnershipTest.kt` is likely root-worthy because it spans many packages.
- Layout-contract and UI-state mapper tests likely belong under `ui/*` if moved.

### 4. Final verification

- The pass only closes if main-source wildcard imports reach zero and full build plus unit tests still pass.
- A direct proof command for zero wildcard imports should be preserved in the final report.

## Recommended Plan Decomposition

1. `25.6-01` — Remove the remaining wildcard imports from all 16 UI-owned main-source files and verify zero total wildcard imports in `app/src/main/java`.
2. `25.6-02` — Apply one more contained extraction pass to `GhostCoreAccessibilityService.kt`, `GhostAccessibilityService.kt`, and `StateCoordinator.kt`, with optional small follow-up in `RuntimeStateStore.kt` or `GhosthandCommandCatalogRoutes.kt` only if clearly justified.
