<decisions_made>

- Keep Ghosthand naming and architecture, but adapt the external MVP's method: fresh current-window reads, action-ready responses, and explicit stale-reference handling
- Add `GET /commands` so on-device agents can discover Ghosthand's commands directly from the app
- Make the local API server concurrent because single-threaded request handling made `/wait` unusable in practice
- The settled screenshot baseline must not require manual root grant or manual screenshot permission grant from users
- Use device-shell orchestration as the authoritative acceptance method for the local-agent route set

</decisions_made>
